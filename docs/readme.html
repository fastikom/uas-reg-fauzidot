Import project - top

After downloading packed file, unzip it. You may follow our video instructions or follow these:

Open Eclipse, click File -> Import... Under Android folder, choose Existing Android Code Into Workspace and click Next. Browse for unzipped folder and choose android folder. Click OK, then click Finish. Before clicking Finish, you may rename project name by clicking SampleAPP under New Project Name tab.

If you use our product in your project, you should import it as library.

In Eclipse, under Package Explorer view, right click your project. Click Properties (should be at the bottom of the right click context). Open Android tab. Under Library box, click Add. Choose our project (if you haven't change project name, should be listed as SampleApp) and click ok.

Usage - top

After import, you may look to the 

com.csform.onlinegallery.sample.SampleAPP.java
code and see how to use this gallery.

You may also run application. Click Run -> Run as -> Android application. Choose connected device or start an emulator.

Here are detailed instructions:

  1. In your activity, create ArrayList of CategoryItem(CategoryItem implements interface Category). 
    ArrayList categories = new ArrayList();
  2. Create as many categories as you need. 
    CategoryItem category = new CategoryItem();
category.setName("CATEGORY 1"); //title of category
category.setResIcon(R.drawable.category1);
//icon of category, if there isn't resID (resID=0) category icon is first picture from list of images.
  3. Create List of ImageItem(ImageItem implements interface Image) 
    ArrayList images = new ArrayList();
  4. Populate images: 
    ImageItem image1 = new ImageItem();
image1.setName("Image1");   //title of image
image1.setPath("http://example.com/image1.jpg"); // path to image, from url or  sdcard uri
image1.setThumbPath("http://example.com/thumbs/image1.jpg); //path to thumb, if path==null full image is used for thumb
  5. Add image1 to images 
    images.add(image1);
  6. Add images to category 
    category.add(images);
  7. Add category to categories 
    categories.add(category);
  8. Choose theme (from Themes.java). Themes.THEME1, Themes.THEME2 or Themes.THEME3 are predefined themes and can be used to determine which theme you would like to use. Pick the one you like.
  9. From your activity, pass categories, title and theme to CategoryGridActivity and start it 
    Intent intent = new Intent(this,CategoryGridActivity.class);
intent.putParcelableArrayListExtra(getString(R.string.categories),categories);
intent.putExtra(getString(R.string.THEME),Themes.THEME1);   //set THEME1, THEME2 or THEME3
intent.putExtra(getString(R.string.TITLE_NAME),"TITLE 1");     //header title
startActivity(intent);

Splash Screen Options - top

Splash screen is set in GalleryCategoryActivity.java file. You can disable it by commenting this line of code, like this (about line 61):

//showSplashScreen();

To change duration of splash screen, edit this line of code (about line 44):

private static final long SPLASH_SCREEN_TIMEOUT_MILLIS = 2000;

to whatever you want. 2000 means 2000 milliseconds, or 2 seconds.

What you can change - top

You can change number of items to be shown in the thumbnail previews. Open Themes.java file and define which of the following you would like to change:

CATEGORY_NUMBER_OF_COLUMNS = {1,2,1};
THUMB_NUMBER_OF_COLUMNS = {2,1,2};

CATEGORY_NUMBER_OF_COLUMNS defines number of columns in category preview, THUMB_NUMBER_OF_COLUMNS defines number of columns in thumbnail preview (images under certain category). These numbers are order by the theme ordering, respectively. So, as from previous code, for CATEGORY_NUMBER_OF_COLUMNS, first number, 1, means category preview in THEME1 should contain 1 column, second number, 2, means category preview in THEME2 should contain 2 columns, and third number, 1, means category preview in THEME3 should contain 1 column.

This also applies for THUMB_NUMBER_OF_COLUMNS (in thumbnail preview, THEME1 has 2 columns, THEME2 has 1 column and THEME3 has 2 columns).

AdMob setup - top

To make your ads work, you just have to provide your Publisher ID taken from here. If you have't registered yet, you can find tutorial how to do it here. After you register, create new site here. Then you should find your newly created Publisher ID at top of the newxt page. Copy it.

Then, in Eclipse, open res folder, open values folder and open tags.xml file. Change PublisherID with your copied PublisherID.

Change Analytics Tracking ID - top

Log in with your Google account here. In the Account column, select the account from the dropdown that you want to add the property to. If you have a lot of accounts, use the search box to help you find the right one. In the dropdown in the Property column, click Create new property. Select Mobile app and fill the rest of fields.

After submiting, copy your tracking ID that is next to name of your newly created site. It should look like with UA-XXXX-Y. Copy it, open Eclipse, find res folder, find values folder and open analytics.xml and paste instead of UA-XXXX-Y.

Important notice - top

If your images are on a public Internet address, don't foget to add permissions to AndroidManifest.xml file.

android.permission.INTERNET

Save file.

In order to “Set wallpeper” option works, you should add this permission:

android.permission.SET_WALLPAPER

To add permision, open AndroidManifest.xml file, click Permissions tab (in the bottom of the screen) and click add. Double click Uses Permission. Find and paste or copy and paste prevous formatted codes.

This app uses UIL (Universal Image Loader) library from Git-Hub under Apache 2.0 licence. You may find their documentation on how to provide URIs for ImageViews to display images. Here is a quick preview:

  1. String imageUri = "http://site.com/image.png"; // from Web
  2. String imageUri = "file:///mnt/sdcard/image.png"; // from SD card
  3. String imageUri = "content://media/external/audio/albumart/13"; // from content provider
  4. String imageUri = "assets://image.png"; // from assets
  5. String imageUri = "drawable://" + R.drawable.image; // from drawables (only images, non-9patch)

This app also uses PhotoView library from Git-Hub under Apache 2.0 licence.

After adding permissions, you should declare Activites in AndroidManifest.xml file. This is a list of activites used in this library:

com.csform.onlinegallery.GalleryCategoryActivity
com.csform.onlinegallery.GalleryThumbActivity
com.csform.onlinegallery.FullScreenGalleryActivity

To declare Activity, open AndroidManifest.xml , click Application tab and click Add under Applications Node tab. Double click Activity. There should be now Attributes for Activity tab, next to Applications Node where you should copy previous formatted codes.
